!Family Chapter 6. The preferences dbox

At present most of the configuration is done through menus. This will change to more resemble the dbox system used under Linux by XFamily. At present just one preferences dbox operates in !Family:

Below are XFamily screenshots, which show the general style towards which the !Family UI is moving. Proper ones and some explanation for the RISC OS !Family preferences will follow soon, but as the interface is shortly going to change, I'm waiting to write a manual page until the interface has stabilised.

The first of the X!Family screenshots shows the tabbed version of the !Family miscellaneous preferences dbox. The !Family style will be tabbed like this, though the icons will be more as the RISC OS style above.

The Names config tab will control how names are displayed on the main window and on printed trees. The XFamily screenshot shows what fields will appear, but again the style will be more RISC OS in !Family.

The dates config tab will control which dates are shown on the tree, and in which style.

At the moment in !Family, you can change most of these options from the iconbar menu which presents submenus from the Choices item. The one option that cannot be changed whilst the program is running is the option to display dates on the tree in either GEDCOM or ISO-8601 format, as a different code library must be loaded for each option and these cannot be changed once loaded. To change this option (and others) you will need to edit the Choices file within the !Family application. A typical Choices file (in fact, the default, as the program is distributed) looks like:

Font:\FHomerton.Medium\ELatin1
PointSize:10
ScreenUseFont:0
ShowFamilyName:-1
ShowDates:-1
SpouseDates:-1
MarrDates:-1
ShowYearOnly:0
ShowTitle:-1
Rotate:-1
DrawScale:1
DrawWidth:4
BadXrefAction:3
ReportDate:ISO
DateLib:ISODateLib
MaxComp:150

To change the way dates are displayed in the main window in !Family, change the DateLib: line to say DEFDateLib instead of ISODateLib, save the file and then run the program.
