A tutorial guide to !Family - a Genealogy database (GEDCOM) editor and browser for the RISC OS desktop. [cite: 1]

1. Introduction

!Family is an editor for Genealogy (Family Tree) Databases working directly with the internationally standardised GEDCOM file format. [cite: 6] It features an intuitive visual interface for browsing and editing families, people, and events directly on a graphical family tree view. [cite: 7]

At present, the software is undergoing a transition from BASIC to C++ to improve speed and memory efficiency. [cite: 8, 14] While currently mature, it is considered alpha release software as the user interface continues to evolve. [cite: 10, 14]

2. The Main Display Window

When started, !Family opens two windows: a tree view and a notes window for the "Current Person." [cite: 15] The display follows these conventions:

  • Color Coding: Males are blue, females are red, and unknown gender is green. [cite: 22]
  • Relationships: Husband and wife are shown one above the other with an equals sign (=). [cite: 25]
  • Navigation: Left-clicking a person makes them the new "current person" and centers the tree on them. [cite: 20]
  • Dates: Birth and death dates appear under each person by default. [cite: 23] A ? indicates an unknown birth date, while a blank indicates an unknown death date. [cite: 24]

2.1 Person Menu

Right-clicking a person opens a menu with several management options: [cite: 29]

  • Edit: Opens a dbox to edit event details like Birth, Christening, and Death. [cite: 29]
  • Marry/Unmarry: Connect or disconnect individuals in the database. [cite: 30, 33]
  • Child: Adds a new person as a descendant of the selected individual. [cite: 35]
  • Older: Swaps sibling display order (this changes the GEDCOM CHIL tags but not birth dates). [cite: 37, 38, 39]
  • Remove: Deletes the person and all their details from the database. [cite: 40]

3. The Person Editing Dbox

This tabbed interface allows for detailed record-keeping for six primary life events: Birth, Christening, Baptism, Death, Cremation, and Burial. [cite: 81]

  • Name Matching: To prevent duplicate entries, you can use F1 (start of name) or F2 (end of name) to search for existing people in the database. [cite: 91, 92]
  • Living Status: A "Living" checkbox is available to flag individuals who are still alive, which helps in suppressing sensitive data during export. [cite: 84, 85]
  • Event Tabs: Each event includes fields for Date, Time, Place, and Site. [cite: 97]
  • Calendars: Supports Gregorian (default) and Julian calendars. [cite: 100, 101]

4. Iconbar Functions

The iconbar menu handles global tasks and reporting: [cite: 60]

  • Print: Attempts to print the tree across multiple pages. [cite: 61]
  • Draw: Produces a RISC OS !Draw file for manual layout manipulation. [cite: 68, 69]
  • Web Pages: Generates HTML files for website publication. [cite: 72, 73] It includes an option to omit details of living persons. [cite: 75]
  • Save: Saves the entire tree in standard GEDCOM format.

Appendix: GEDCOM Files

GEDCOM (Genealogical Data Communication) is the international standard for genealogical data. [cite: 126] Files are plain ASCII text. [cite: 130] Each line follows a specific grammar:

level [@id@] tag value

For example, a family unit is defined by FAM tags that cross-reference individual (INDI) records for husbands, wives, and children. [cite: 134, 139]

!Family aims to preserve all tags, even those it does not currently support, to ensure no data is lost when saving. [cite: 149, 154]
